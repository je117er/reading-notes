- a good metric should have the following properties:
	- easy to use
	- minimum human involvement
	- high accuracy
	- min time wastage
- metric should answer the following questions:
	- type of metric
	- reason for selection
	- source of aggregated data
	- mathematical basis/formula for the calculation
	- how often will it be calculated
	- how often will it be used
### data collection process
good points to keep in mind:
- involve stakeholders to interpret data
- avoid measure everything: not every metric is created equal
- no collecting useless data that doesn't serve any higher goal

## types of lean metrics
- financial metrics
	- cost-related metrics
	- revenue-related metrics
- behavioral metrics
	- commitment-based
	- comm-based
	- cooperation-based
- core process metrics
	- core-process metrics (development cycle time, time to market, product life cycle profitability)
	- result metrics: ontime delivery
	- productivity metrics: why do these sound like they could overlap with core-process ones?
